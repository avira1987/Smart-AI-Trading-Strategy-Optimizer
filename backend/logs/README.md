# لاگ‌های بک‌تست

این پوشه شامل لاگ‌های کامل بک‌تست می‌شود که برای رفع اشکال استفاده می‌شوند.

## فایل‌های لاگ

1. **backtest.log** - لاگ‌های کامل بک‌تست شامل:
   - اطلاعات شروع و پایان هر بک‌تست
   - جزئیات استراتژی (شرایط ورود/خروج)
   - اطلاعات داده‌های دریافتی
   - مراحل اجرای بک‌تست (محاسبه اندیکاتورها، تولید سیگنال‌ها، اجرای معاملات)
   - نتایج نهایی
   - خطاهای کامل با traceback

2. **api.log** - لاگ‌های عمومی API و مدیریت درخواست‌ها

## نحوه استفاده

1. یک بک‌تست را اجرا کنید
2. بعد از اتمام (یا خطا)، فایل `backtest.log` را باز کنید
3. آخرین بک‌تست را پیدا کنید (جستجو برای Job ID)
4. تمام لاگ‌ها را از `========== شروع بک‌تست` تا `========== بک‌تست تکمیل شد` کپی کنید
5. لاگ‌ها را برای بررسی ارسال کنید

## نکات مهم

- لاگ‌ها به صورت چرخشی ذخیره می‌شوند (حداکثر 10MB هر فایل، 5 نسخه backup)
- لاگ‌ها شامل تمام جزئیات فارسی و انگلیسی هستند
- در صورت خطا، traceback کامل ذخیره می‌شود

## جستجوی سریع در لاگ

برای پیدا کردن آخرین بک‌تست:
```
========== شروع بک‌تست برای Job ID:
```

برای پیدا کردن خطاها:
```
❌❌❌ بک‌تست با خطا مواجه شد
```
